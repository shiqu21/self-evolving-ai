## 2. 产品定义

### 2.1 产品目标

| 目标 | 描述 |
|------|------|
| **目标1：自主进化** | 系统通过5阶段进化循环（技能优化→工具描述→系统提示→代码优化→持续循环）实现自我进化，每次迭代成本$2-10，无需GPU训练 |
| **目标2：自动运行** | 基于核心循环OPERATE→DETECT→ANALYZE→ENCODE→VERIFY自动执行，配合定时任务实现无人值守运行 |
| **目标3：质量保障** | 通过8阶段质量管道（WRITE→TYPECHECK→TEST→LINT→CRITIQUE→REFINE→ESCALATE→ARBITER）和人工审查PR机制确保输出质量 |

| US3 | 作为系统，我需要三阶段记忆架构（感觉缓冲区→海马编码→皮层整合）来积累和检索知识 | P0 |
| US4 | 作为质量评审员，我希望通过多代理评审机制（事件监控器→事件分析师→质量评审员→质量仲裁者）确保代码质量 | P1 |
| US5 | 作为运维人员，我希望系统配置定时任务实现自动运行，无需人工干预 | P1 |
| US6 | 作为系统，我希望通过GEPA算法优化提示词和技能，实现越用越聪明 | P2 |

### 4.2 核心模块说明

| 模块 | 职责 | 关键类/函数 |
|------|------|------------|
| `core/evolution_loop.py` | 5阶段进化循环引擎 | `EvolutionLoop.run()`, `operate()`, `detect()`, `analyze()`, `encode()`, `verify()` |
| `core/quality_pipeline.py` | 8阶段质量管道 | `QualityPipeline.execute()`, 各阶段处理器 |
| `agents/orchestrator.py` | 任务编排与代理协调 | `Orchestrator.delegate()`, `Orchestrator.manage_dependencies()` |
| `agents/event_monitor.py` | 错误扫描与事件报告 | `EventMonitor.scan_sessions()`, `EventMonitor.create_report()` |
| `agents/event_analyst.py` | 根因分析与规则生成 | `EventAnalyst.analyze()`, `EventAnalyst.generate_proposal()` |
| `agents/code_reviewer.py` | 代码质量评审 | `CodeReviewer.review()` |
| `agents/arbiter.py` | 质量仲裁决策 | `Arbiter.make_decision()` |
| `agents/skill_writer.py` | 技能模块生成 | `SkillWriter.generate()`, `SkillWriter.fill_gap()` |

### 4.4 技术选型

| 组件 | 选型 | 理由 |
|------|------|------|
| 定时任务 | Python schedule / APScheduler | 简单可靠，无需额外依赖 |

| 记忆存储 | SQLite数据库 | 结构化查询能力 |
| 日志 | Python logging | 内置支持 |
| 配置 | YAML/JSON | 易于修改和版本控制 |
